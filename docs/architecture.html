System Reference · inventory-management

Factory Inventory Management — Architecture

Full-stack demo: a Vue 3 SPA talking to a FastAPI backend that serves filtered, in-memory JSON fixtures. No database, no auth layer — every request is a pure read over static data loaded at startup.

System Overview

Client Browser SPA served by Vite dev server. Router handles 6 views client-side. localhost:3000
Frontend Vue 3 + Vite Composition API. Global filter state via module-singleton composables. Axios client in api.js. vue-router · axios
Backend FastAPI 14 read-only endpoints. Pydantic response models. CORS open to all origins. localhost:8001
Data JSON fixtures 7 files in server/data/ loaded into memory by mock_data.py at import. in-memory · no persistence

Request path: filter change → axios GET with query params → FastAPI list comprehension → JSON

Data Flow — Filter Pipeline

FilterBar.vueclient/src/components/
Four dropdowns (period, warehouse, category, status) two-way bound to singleton refs in useFilters.
useFilters.jsclient/src/composables/
Module-level ref() singletons hold filter state. getCurrentFilters() remaps location→warehouse and period→month for the API.
View componentclient/src/views/*.vue
Watches filter refs; on change calls api.getOrders(getCurrentFilters()) etc. Stores response in local ref, derives display data via computed.
api.jsclient/src/
Builds URLSearchParams, dropping any filter set to 'all'. Sends axios GET to http://localhost:8001/api/*.
main.pyserver/
Endpoint reads query params → apply_filters() does field-equality match, filter_by_month() handles YYYY-MM and Q1-2025…Q4-2025 via QUARTER_MAP.
mock_data.pyserver/
Source lists (inventory_items, orders, …) loaded once from server/data/*.json at import. Filtering is a list comprehension over these — no I/O per request.

Notable

State management. No Pinia/Vuex — shared state lives as module-level ref() exports in composables, giving a singleton per import graph.
No persistence. All data is loaded from server/data/*.json into Python lists at startup. Restarting the server resets everything.
Frontend/backend drift. api.js defines getTasks/createTask/deleteTask/toggleTask and createPurchaseOrder/getPurchaseOrderByBacklogItem, but no matching routes exist in server/main.py. PurchaseOrder Pydantic models exist unused.
Bypassed abstraction. Reports.vue calls axios directly to /api/reports/* instead of going through api.js.
Orphaned view. Backlog.vue exists and works but has no route in main.js.
